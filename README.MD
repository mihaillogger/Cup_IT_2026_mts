# Предсказание высоты зданий для радиопланирования | MTS Cup IT 2026

## О бизнес-задаче
**Данный проект — абсолютный победитель (1-е место) Всероссийского кейс-чемпионата MTS Cup IT 2026 в треке «Аналитика данных» (команда «Кронверкские псы»).**
Проект разработан в рамках финала кейс-чемпионата **MTS Cup IT 2026**. Основная цель — создать production-ready пайплайн для предсказания физической высоты зданий (`target_height`) на основе разрозненных геопространственных и кадастровых данных. Точные данные о высоте критически важны для радиопланирования, расчета покрытия 4G/5G и оптимизации размещения базовых станций.

**Достигнутый результат (OOF K-Fold CV):**
* **MAE:** 0.5136 м (средняя ошибка предсказания — около 50 см)
* **RMSE:** 2.5785 м
* **R²:** 0.9659

## Архитектура решения и Feature Engineering

Пайплайн решает фундаментальную проблему геоаналитики: разорванные полигоны и несовпадение ID в разных базах данных. Мы отказались от "слепого" джойна и реализовали 4-ступенчатую архитектуру:

### 1. Внутренняя сборка и метризация (Ноутбук 01)
* Перевод координат из GPS (EPSG:4326) в плоскую метрическую систему (EPSG:32636) для честного расчета площадей и дистанций.
* Динамическая фильтрация геометрического мусора (по 10-му перцентилю площади).
* Присоединение "оторванных" пристроек к основным зданиям (`sjoin_nearest` с допуском 3 метра) и их "расплавка" (`dissolve`), при которой этажность основного здания агрегируется по максимуму.

### 2. Графовое объединение комплексов (Ноутбук 02)
* Пространственное пересечение Источника А и Источника Б с расчетом метрики **IoU (Intersection over Union)**.
* Использование графовых алгоритмов (`NetworkX`) для поиска связных компонент (connected components). Здания с пересечением IoU > 0.2 объединялись в единые архитектурные комплексы (`complex_id`).

### 3. Алгоритм «Альфа/Бета» (Ноутбук 02)
* Внутри каждого `complex_id` алгоритм автоматически выделяет **Альфу** (главное здание с максимальной площадью) и **Беты** (пристройки).
* Альфа забирает максимальную известную этажность, Беты — минимальную. Этот алгоритм позволил спасти и корректно распределить более 30 000 значений этажности, которые иначе были бы утеряны при слиянии.

### 4. Динамический пространственный KNN (Ноутбук 03)
* Для учета локального "высотного регламента" вычисляется средняя высота 5 ближайших соседей. 
* Во избежание утечки данных (Target Leakage), расчет изолирован строго внутри циклов K-Fold кросс-валидации.

## Выбор ML-модели

В ходе экспериментов сравнивались CatBoost, XGBoost и LightGBM. Матрица корреляций предсказаний показала сходство алгоритмов на уровне **>0.99**. 
Руководствуясь метриками ROI, скоростью инференса и потреблением памяти при обработке федеральных геоданных, в production внедрен **LightGBM Regressor** с функцией потерь $L_1$ (MAE) для робастности к кадастровым аномалиям. Итоговое предсказание формируется через ансамблирование 5 фолдов.

## Инструкция по запуску
Проект имитирует реальный последовательный ETL-процесс. Для воспроизведения результатов выполните следующие шаги:

1. **Клонируйте репозиторий и установите зависимости:**
   ```bash
   git clone <URL_репозитория>
   cd <имя_папки>
   pip install -r requirements.txt
   ```

2. **Подготовьте исходные данные:**
Разместите файлы от организаторов (cup_it_example_src_A.csv и cup_it_example_src_B.csv) в директорию data/raw/.

3. **Запустите пайплайн:**
Последовательно запустите все ячейки в Jupyter-ноутбуках:

* Шаг 1: 01_EDA_and_Insights.ipynb

* Шаг 2: 02_Merging_and_Feature_Engineering.ipynb

* Шаг 3: 03_ML_3_D_Model.ipynb

4. **Результат:**
Финальный файл с предсказаниями (FINAL_SUBMISSION_LGBM_3D.csv) и интерактивная 3D-визуализация карты (в формате .html) будут автоматически сохранены в папку submissions/.
